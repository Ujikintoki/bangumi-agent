# Tool7: 获取人物吐槽
## 1. 核心意图
[大模型调用此工具的意图]
获取某位现实人物（声优、导演、作者等）的社区吐槽箱，用于分析观众对该人物的评价、人气或讨论。**注意**：人物的基本信息（如职业、代表作）属于 LLM 常识，无需通过此工具获取——此工具仅提供 LLM 不知道的用户生成内容。

## 2. 聚合的 GET API 路由
1. `[GET] /p1/persons/{personID}/comments` （获取人物吐槽箱）

## 3. Pydantic 入参建议 (Input)
[大模型调用此工具时需要传入的参数。]
* `person_id` (int): 人物纯数字 ID。通常由上游搜索工具透传。
* `limit` (int, 可选，默认 20，最大值 50): 返回吐槽条数。通过 `Field(le=50)` 硬性限流。

## 4. 核心 JSON 字段提取 (Output 契约)
**API 1: `/p1/persons/{personID}/comments`**
*(极致瘦身：与角色吐槽箱采用完全相同的压缩策略，抛弃所有元数据)*

* `comments` (list[str]): 纯文本吐槽列表。
  * **格式定义**：`"[{likes}赞] {content}"`
  * **likes 计算**：`reactions` 数组中所有 `reaction.users` 长度的总和。
  * **reply 处理**（可选）：若 `replies` 非空，取第一条回复：`"[{likes}赞] {content} —— 回复: {reply_content}"`。
  * **示例**：`"[128赞] 梶裕贵在这部作品里的演技真的封神了"`

## 5. 业务防御与特殊说明
* **噪音过滤 (Noise Filtering)**：
  * 规则 1：剔除 `comment` 长度小于 4 个字符的短评。
  * 规则 2：剔除纯数字、纯日期、纯标点符号组成的评论。
* **数据截断**：通过 `limit` 参数控制（默认 20，最大 50）。超出时在末尾追加 `"... 还有 {N} 条评论"`。
* **人物名提示**：在返回 JSON 顶部增加 `"person": "{人物名}"` 字段。
* **404 异常熔断**：捕获 404，转译为 `"未找到 ID 为 {personID} 的人物，可能已被删除或合并"`。
* **空评论处理**：返回 `{"person": "...", "comments": [], "hint": "该人物暂无社区吐槽"}`。
* **通用异常熔断**：同角色吐槽，捕获 HTTP 异常转译为自然语言，绝不抛 Exception。
